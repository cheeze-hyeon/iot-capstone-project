# 알록 리필 스테이션 — 사용자 워크플로우

리필 스테이션 키오스크(`pos.html`)의 화면 흐름과 각 단계에서의 하드웨어 동작을 정리한 문서.

## 전체 흐름 요약

```
[STEP 1] 카드 태그
   ↓ (RFID 태그)
[STEP 2] 회원 인식 + 제품 선택
   ↓ (제품 카드 클릭)
[STEP 3] 용기 준비
   ↓ (빈 용기 올리고 버튼 누름 → tare 측정)
[STEP 4] 리필
   ↓ (내용물 담고 버튼 다시 누름 → 리필 완료)
[STEP 5] 결제
   ↓ (결제 완료 버튼)
[STEP 6] 완료 (탄소 절감량 표시)
   ↓ (처음으로 버튼 → STEP 1로 리셋)
```

화면 상단의 STEP 바(`STEP n · 단계명 / 6`)가 항상 현재 단계를 보여준다.

## 단계별 상세

### STEP 1. 카드 태그 (`CARD` 화면, 서버 상태 `WAIT_CARD`)
- 초기 상태. LED는 꺼져 있음.
- 화면: "리더기에 카드를 태그해주세요" + 카드 아이콘 + "카드 태그 대기 중" 배지(빨간 점).
- 손님이 RFID 카드를 RC522 리더기에 태그하면:
  - 아두이노가 "띠-로롱" 효과음(beepCardTag) 재생
  - 빨간 LED 켜짐
  - 서버가 `CARD_TAGGED` 이벤트를 브로드캐스트, 서버 상태 → `IDLE`

### STEP 2. 회원 인식 + 제품 선택 (`SELECT` 화면)
- "{회원이름}님 환영합니다!" 큰 제목 + "어떤 제품을 리필하시겠어요?" 부제
- 4종 제품(세탁 세제 / 섬유 유연제 / 샴푸 / 주방 세제) 카드 그리드, 각 g당 가격 표시
- 카드 UID → 이름 매핑은 `pos.html`의 `MEMBERS` 객체에서 관리 (예시: `BB1CAA00` → "최주현", 미등록 카드는 "고객")
- 제품 카드를 클릭하면 STEP 3으로 이동 (제품 선택만 클라이언트 상태이며 서버에는 별도 신호를 보내지 않음)

### STEP 3. 용기 준비 (`PLACE` 화면, 서버 상태 `IDLE`)
- 선택한 제품을 칩(아이콘+이름)으로 표시
- "빈 용기를 저울 위에 올려주세요. 올려놓으셨으면 버튼을 눌러주세요."
- 저울 실시간 무게를 큰 숫자로 표시 (안정/측정 중 색상 구분)
- "준비 중 — 버튼을 눌러주세요" 배지(빨간 점)
- "제품 다시 선택" 버튼 → STEP 2로 돌아감 (서버 상태는 유지, 리셋 안 함)
- **물리 버튼**을 누르면:
  - 현재 무게를 `tare`로 저장
  - 파란 LED 켜짐, 서보모터 열림(디스펜서 게이트 오픈)
  - 서버 상태 → `READY`, 화면은 자동으로 STEP 4로 전환

### STEP 4. 리필 (`REFILL` 화면, 서버 상태 `READY`)
- "원하는 만큼 용기에 담아주세요. 화면을 보면서 양을 조절하세요. 다 담으셨으면 버튼을 다시 눌러주세요."
- 실시간 리필량 = 현재 무게 − tare (큰 숫자로 표시)
- 예상 가격 = 리필량 × g당 단가
- "리필 가능" 배지(파란 점)
- 화면의 "리필 완료" 버튼 **또는** 물리 버튼을 다시 누르면:
  - 서보모터 닫힘(디스펜서 게이트 클로즈)
  - "도-미-솔" 경쾌한 효과음(beepDone) 재생
  - `REFILL_DONE` 이벤트 브로드캐스트 → 이 시점의 리필량을 `finalRefill`로 캡처 → STEP 5로 이동

### STEP 5. 결제 (`CHECKOUT` 화면)
- 영수증 형태: 제품명/아이콘, 리필량(g), 단가, 결제 금액(리필량 × 단가)
- "결제 완료" 버튼 → STEP 6

### STEP 6. 완료 (`DONE` 화면)
- 잎사귀 아이콘 + "결제가 완료되었습니다. 일회용 포장을 줄여서"
- 탄소 절감량 = 리필량 × 제품별 `co2PerGram` (g CO2 단위로 표시)
- "처음으로" 버튼 → 모든 클라이언트 상태 초기화, 서버에 `reset` 전송 → STEP 1(`WAIT_CARD`)로 복귀, LED 꺼짐

## 물리 버튼의 이중 역할

실물 버튼 하나가 두 가지 역할을 한다:
1. **STEP 3 (IDLE)**: 누르면 tare 측정 + 리필 시작 (화면 "리필 완료" 버튼과 별개의 트리거)
2. **STEP 4 (READY)**: 누르면 리필 완료 신호 (화면의 "리필 완료" 버튼과 동일한 효과)

서버는 `station_state`에 따라 같은 `BUTTON_PRESSED` 신호를 다르게 해석한다.
